Bug Reports, Suggestions, and Info!

Reporting a Bug

If you find a bug, which is honestly quite likley, please do the following.

1. Right click the page and click "inspect". A new page will open up.
2. Click "Console". Doing so will reveal all errors from the code.
3. Send me a screenshot of the console errors, as well as a detailed report of what the bug is and what is required to reproduce it.

Technical Information

All of the backend data for this project is stored on a firebase realtime database. I did this because 1. it's free and 2. it's quite a bit easier to use than creating my own backend from scratch. All of the data, besides passcodes, is stored unencrypted on this database. There are read and write permission systems in place based on firebase auth.

I don't trust myself to not mess up security, so I do not allow the user to choose their own passcode. Everyone would probably use the same passcode they use for every other account they own, and if I mess up and these passcodes are somehow aquired by a malicious third party, it would be bad. Very bad.

There is no reason any passcodes should be leaked because they are hashed before they even make it to the server. BUT. The messages are not encrypted, and the read/write permissions are not watertight. There is probably some exploit I didn't think of that would allow people to read messages in chats they aren't a part of. So please, do not share your deepest darkest secrets over this platform. Assume it could be read by someone else.
